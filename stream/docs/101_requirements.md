# マルチコンテンツ・ストリーミングサービス 要件定義書

## 1. サービスの目的と概要
動画、画像、360度動画など、多様なデジタルコンテンツをシームレスに提供し、洗練されたユーザーインターフェースと高いスケーラビリティを備えた本格的なマルチコンテンツ・ストリーミングプラットフォームを構築する。

## 2. システムの提供機能 (Functional Requirements)

### 2.1 エンドユーザー向け機能 (User Facing)
- **ユーザー認証・アカウント管理**
  - サインアップ・ログイン機能（メールアドレス・パスワード、またはOAuth等）。
  - JWT等を用いたセキュアなセッション管理。
  - プロフィール管理（将来の複数プロフィール対応のための基盤設計）。
- **コンテンツ閲覧・検索**
  - トップページでの魅力的なコンテンツ表示（動画、画像、VRが入り乱れるグリッド/カルーセル）。
  - マウスオーバー/フォーカス時のプレビュー表示（動画は自動再生、画像はズーム等）。
  - 種別（Video, Image, VR）やジャンル別の一覧表示、キーワード検索。
- **マルチメディア再生・閲覧機能**
  - **動画 (Video)**: HLSを用いたAdaptive Bitrate Streaming。
  - **360度動画 (360° VR)**: ブラウザ上での視点操作。
  - **画像ギャラリー (Image Gallery)**: 高解像度ビューアー、スライドショー機能。
- **パーソナライズ（初期設計考慮事項）**
  - 視聴履歴の記録・表示（どこまで見たかのレジューム再生）。
  - お気に入り（マイリスト）機能。

### 2.2 管理者向け機能 (Admin / Backoffice)
- **コンテンツ管理 (CMS)**
  - 作品（タイトル、あらすじ、リリース年等）の登録・編集・公開・非公開。
  - 画像データ（サムネイル、ポスター）のアップロードと紐付け。
  - 動画データ（HLSマニフェストパス、セグメントディレクトリ）の紐付け。
- **メタデータ管理**
  - ジャンルの登録・編集、出演者・監督などのタグ情報の管理。

## 3. 非機能要件 (Non-Functional Requirements)

### 3.1 パフォーマンス・可用性
- **シーク時の爆速レスポンス**: 再生時の待ち時間やシーク時のバッファリングを極小化する。
- **エッジキャッシュの最大活用**: 静的アセット（画像、動画セグメント）はS3互換ストレージをオリジンとし、CDN（Cloudflare等）を経由してエッジから配信する。バックエンドへの負荷を最小限に抑える。
- **UI描画速度**: BFF（Backend for Frontend）層や MongoDB の最適化を用い、クライアントへのメタデータ応答をミリ秒単位で行う。

### 3.2 拡張性・柔軟性
- **マイクロサービス・アーキテクチャ**: 各機能（認証、メタデータ、カタログ、ストリーミング）を独立したサービスとして構築し、将来的な負荷増加に対してスケールアウトしやすくする。
- **多様なコンテンツへの適応**: RDBに加えNoSQL（MongoDB等）を併用することで、将来の未知のフォーマット（電子書籍、ゲーム等）のメタデータにも柔軟に対応可能なデータ構造とする。

### 3.3 セキュリティ・運用保守性
- **IDの難読化**: DB内部で扱うプライマリキー（UUID/連番等）と、ユーザーUI上でURLなどに露出させるID（Short ID: 例 `nano_id` 等）を分離し、データ構造の推測やスクレイピングを防ぐ。
- **インフラのコンテナ化**: 全てのサービスはDocker等でコンテナ化され、デプロイやローカル環境構築が容易に行えること。
