# 運用マニュアル：データ同期

本文書では、PostgresSQLマスターデータベースとMongoDBカタログストア間のデータ同期方法について説明します。

## 背景
システムはCQRSパターンを採用しており、以下の役割分担となっています。
- **PostgreSQL**: メタデータの正（Source of Truth）。
- **MongoDB**: カタログサービス用の非正規化ストア。

管理API経由で変更が行われた際はオンラインでデータ同期されますが、初期セットアップやデータの整合性確認（Reconciliation）のためにバッチ処理が必要です。

## 一括同期 (Batch Synchronization)

Go言語ベースの同期バッチがDockerサービスとして利用可能です。

### バッチの実行方法
PostgreSQLからカタログサービスへ全コンテンツを同期するには、以下のコマンドを実行します。

```bash
docker compose run --rm sync-batch
```

このバッチは以下の処理を行います：
1. PostgreSQLの `contents` テーブルからすべての `short_id` を取得します。
2. 5つのワーカーを使用して、各アイテムをMongoDBに並行同期します。

### 効率性
バッチはワーカープールを使用して複数のアイテムを並列に同期するため、コンテンツライブラリが増大しても高いパフォーマンスを維持します。

## インフラの詳細
アセット（ポスター、動画）は **MinIO** を介して提供されます。
`assets` バケットは、`./db/tests/minio/assets` を `/assets-seed` にマウントし、サイドカー経由でコピーすることで事前にデータが投入されます。
