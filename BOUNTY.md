- **リポジトリ**: k-wa-wa/pechka
- **OWNER**: k-wa-wa（レビュー・意思決定者）
- **ME**: bot-wa-wa（実装・対応担当エージェント）

## 自動開発タスク

以下の手順で GitHub 上のタスクを処理する。

### 0. 未完了作業の確認（最初に必ず実施）

セッション開始時に、前回の作業が中断していないか確認する。

- 現在のブランチが master 以外の場合、そのブランチに対応する PR があるか確認する
- PR がない場合は、ブランチの内容を把握して PR を作成し、対応する issue にリンクを貼り、OWNER にレビュー依頼する
- PR がある場合は「1. PR の確認・対応」に進む

### 1. PR の確認・対応

GitHub リポジトリにおいて、以下を満たす PR があるか確認する。

- open であること
- OWNER の確認待ち状態ではないこと（例: レビュー依頼済みで応答待ち、「確認待ち」コメントがある等）
- ME が着手可能なタスクであること

該当 PR があった場合:
- PR の内容を把握して対応する
- タスク完了時には、PR 上で再度レビュー依頼をし、OWNER にアクションを求める

### 2. Issue の確認・対応（対応すべき PR がない場合）

GitHub の以下の条件を満たす issue があるか確認する。

- open であること
- OWNER の確認待ち状態ではないこと
- ME が着手可能なタスクであること

該当 issue があった場合:
- issue の内容を把握して対応する
- issue が複数タスクに分かれる場合は、sub issue を作成してその中で対応する
- sub issue を作成した場合は、親 issue に sub issue へのリンクを貼る
- 開発 ready であれば、実装を行い、PR を作成し OWNER にレビュー依頼する
  - PR 作成時は必ず対応する issue 番号を本文に記載する（例: `Closes #2`）
- issue の対応が完了した場合は、issue 上で完了したことを報告し、OWNER にアクションを求める
