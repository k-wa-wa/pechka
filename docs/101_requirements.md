# ストリーミングサービス 要件定義書

## 1. サービスの目的と概要

個人利用の動画ストリーミングサービス。Bluray ディスクからコンテンツを取り込み、ブラウザからシームレスにストリーミング再生できる環境を構築する。マルチスクリーン・マルチ品質対応の HLS 配信を基盤とし、**性能を最優先**に設計する。

## 2. システムの提供機能 (Functional Requirements)

### 2.1 コンテンツ閲覧機能

- **コンテンツ一覧**: 登録済みコンテンツの一覧表示（動画、画像、VR）。
- **コンテンツ検索**: キーワード・タグによる絞り込み（Elasticsearch 使用）。
- **動画再生**:
  - HLS Adaptive Bitrate Streaming（ABR）による品質の自動切り替え。
  - 手動での品質選択（1080p / 720p / 480p / 音声のみ）。
  - 360度 VR 動画はブラウザ上での視点操作。
- **画像ギャラリー**: 高解像度ビューアー。

### 2.2 管理機能

- **コンテンツ管理 (CMS)**: タイトル・説明・タグの編集、公開/非公開の切り替え。
- **サムネイル管理**: 自動生成サムネイルの確認・差し替え。
- 認証・認可は不要（個人利用のため）。

### 2.3 Bluray 取り込みパイプライン

- **Phase 1 - Extract（物理デバイス依存）**: Bluray ディスク → MakeMKV → NFS ストレージへの MKV 保存。
  - 1 枚のディスクから複数タイトルを個別の MKV ファイルとして抽出。
  - 取り込み完了後も MKV は NFS に永続保持（再変換の容易さを確保）。
- **Phase 2 - Transform（NFS ファイル操作）**: MKV → ffmpeg → 複数品質の HLS 生成。
  - 映像+音声: 1080p / 720p / 480p（ABR マスタープレイリスト付き）。
  - 音声のみ: AAC HLS ストリーム。
  - 低画質バリアントも作成（帯域幅の少ない環境向け）。
  - HLS セグメント・マニフェストを MinIO にアップロード。
- **Phase 3 - Load（カタログ登録）**: PostgreSQL 登録 → MongoDB 同期 → Elasticsearch インデックス登録 → サムネイル生成。

### 2.4 データモデル

- **discs**: Bluray ディスク 1 枚を管理するエンティティ。
- **contents**: 各コンテンツ（タイトル）を管理。1 ディスクから複数コンテンツを登録可能。
- **video_variants**: コンテンツごとの動画バリアント（品質別 HLS）を管理。
- **assets**: サムネイル・ポスター等のメディアアセット。

## 3. 非機能要件 (Non-Functional Requirements)

### 3.1 パフォーマンス（最優先）

- **シーク高速化**: HLS セグメントは MinIO から直接配信。Nginx キャッシュによるエッジキャッシュを最大活用し、シーク時のバッファリングを極小化する。
- **メタデータ API 応答速度**: カタログ API は ms 単位で返却。MongoDB（読み取り最適化）+ API 層のインメモリキャッシュを活用する。
- **検索速度**: Elasticsearch によるリアルタイム全文検索。
- **トランスコードスループット**: ffmpeg の並列処理、可能であれば GPU（h264_nvenc）エンコードを活用し、取り込み時間を短縮する。
- **ABR プレーヤー**: クライアントのネットワーク状況に応じた品質自動切り替えで、途切れのない再生体験を提供する。

### 3.2 シンプルさ・運用性

- 認証・認可は不要（アクセス制御はインフラ層または LAN 限定運用で実施）。
- 全てのコンポーネントは Docker コンテナ化し、K8s (k3s) でデプロイする。
- NFS・MinIO・PostgreSQL に障害があっても他サービスへの影響を局所化する。

### 3.3 データ保全

- Bluray から変換した MKV ファイルは NFS 上に永続保持（再変換のために必要）。
- HLS セグメント（.ts, .m3u8）は MinIO に保持（配信元）。
- PostgreSQL はコンテンツメタデータの Single Source of Truth（信頼できる情報源）。
- MongoDB は PostgreSQL のキャッシュ用読み取りレプリカ（Benthos により同期）。
- Elasticsearch は全文検索インデックス（PostgreSQL から定期 or イベント駆動で同期）。

### 3.4 拡張性（将来対応）

- チャプターマーカー（VideoTimestamps）機能: 後フェーズで対応。
- プレイリスト・グルーピング機能: 後フェーズで対応。
- シリーズ/エピソード概念: 後フェーズで対応。
- 視聴履歴・レジューム再生: 後フェーズで対応。
